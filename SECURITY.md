# سياسة الأمان والخصوصية / Security & Privacy Policy\n\n## 1. الالتزام بالأمان / Security Commitment\n\nفي ProfitPilot، نعتبر أمان بيانات عملائنا أولويتنا الأساسية. نحن ملتزمون بأعلى معايير الأمان والخصوصية.\n\nAt ProfitPilot, we consider the security of our customers' data to be our top priority. We are committed to the highest standards of security and privacy.\n\n---\n\n## 2. معايير التشفير / Encryption Standards\n\n### البيانات أثناء النقل / Data in Transit\n- **HTTPS/TLS 1.2+**: جميع الاتصالات مشفرة باستخدام TLS 1.2 أو أحدث\n- **Certificate Pinning**: التحقق من شهادات SSL/TLS\n- **Perfect Forward Secrecy**: مفاتيح جلسة فريدة لكل اتصال\n\n### البيانات في الراحة / Data at Rest\n- **AES-256 Encryption**: تشفير قوي للبيانات المخزنة\n- **Database Encryption**: تشفير قاعدة البيانات بالكامل\n- **Secure Key Management**: إدارة آمنة للمفاتيح التشفيرية\n\n---\n\n## 3. إدارة المصادقة / Authentication Management\n\n### OAuth 2.0\n- **معيار صناعي**: استخدام OAuth 2.0 لجميع التكاملات\n- **لا مشاركة كلمات المرور**: لا نطلب أبداً كلمات المرور\n- **التوكن المحدود**: توكنات محدودة الصلاحيات والمدة\n\n### إدارة التوكن\n```\n- تخزين آمن في localStorage مع تشفير\n- تحديث تلقائي قبل انتهاء الصلاحية\n- حذف فوري عند تسجيل الخروج\n- تسجيل جميع محاولات الوصول\n```\n\n---\n\n## 4. صلاحيات الوصول / Access Permissions\n\n### مبدأ أقل صلاحية / Principle of Least Privilege\n\nنطلب فقط الصلاحيات الضرورية:\n\n#### Salla\n```\n- products:read (قراءة المنتجات)\n- orders:read (قراءة الطلبات)\n- inventory:read (قراءة المخزون)\n```\n\n#### Zid\n```\n- products:read\n- orders:read\n- inventory:read\n```\n\n#### Shopify\n```\n- read_products\n- read_orders\n- read_inventory\n```\n\n#### WooCommerce\n```\n- Basic Auth (Consumer Key + Secret)\n- Read-only API keys\n```\n\n#### Meta Ads\n```\n- ads_read (قراءة الحملات)\n- read_insights (قراءة المقاييس)\n- ❌ لا توجد صلاحيات كتابة\n```\n\n#### Google Ads\n```\n- adwords.readonly (قراءة فقط)\n- ❌ لا توجد صلاحيات كتابة\n```\n\n---\n\n## 5. حماية حسابات الإعلانات / Ad Account Protection\n\n### لماذا لن يتم حظر حسابك؟\n\n1. **صلاحيات القراءة فقط**\n   - لا يمكننا تعديل أي شيء\n   - لا يمكننا صرف الميزانيات\n   - لا يمكننا حذف الحملات\n\n2. **نشاط طبيعي**\n   - الطلبات محدودة ومنتظمة\n   - لا نحاكي سلوك إنساني\n   - نتبع معايير API الرسمية\n\n3. **معايير صناعية**\n   - نستخدم نفس الطريقة التي تستخدمها Supermetrics و Hootsuite\n   - معتمدة من Meta و Google\n   - آمنة 100%\n\n### أفضل الممارسات / Best Practices\n\n```\n✅ استخدم حسابات إعلانية منفصلة للاختبار\n✅ راجع الأذونات المطلوبة قبل الموافقة\n✅ قطع الاتصال إذا لم تعد بحاجة إليه\n✅ استخدم كلمات مرور قوية لحسابات Meta و Google\n❌ لا تشارك بيانات اعتماد حسابك مع أحد\n❌ لا تستخدم حسابات شخصية للإعلانات\n```\n\n---\n\n## 6. تخزين البيانات / Data Storage\n\n### أين يتم تخزين البيانات؟\n\n- **قاعدة البيانات**: Supabase PostgreSQL (مشفرة)\n- **التخزين السحابي**: Supabase Storage (للصور)\n- **التوكنات**: localStorage مشفر (المتصفح)\n\n### مدة الاحتفاظ بالبيانات / Data Retention\n\n- **بيانات المنتجات**: طالما كان الاتصال نشطاً\n- **بيانات الطلبات**: حسب سياسة الاحتفاظ المختارة\n- **بيانات الحملات**: حسب سياسة الاحتفاظ المختارة\n- **السجلات**: 90 يوماً\n\n---\n\n## 7. حماية البيانات الشخصية / Personal Data Protection\n\n### معايير الامتثال / Compliance Standards\n\n- **GDPR**: الامتثال الكامل لقانون حماية البيانات الأوروبي\n- **CCPA**: الامتثال لقانون خصوصية المستهلك في كاليفورنيا\n- **Local Laws**: الامتثال للقوانين المحلية في السعودية والإمارات\n\n### حقوق المستخدم / User Rights\n\n```\n✅ الحق في الوصول إلى بيانات\n✅ الحق في تصحيح البيانات\n✅ الحق في حذف البيانات\n✅ الحق في نقل البيانات\n✅ الحق في سحب الموافقة\n```\n\n---\n\n## 8. مراقبة الأمان / Security Monitoring\n\n### التسجيل والمراقبة / Logging & Monitoring\n\n```\n- تسجيل جميع محاولات الوصول\n- مراقبة النشاط المريب\n- تنبيهات فورية للأنشطة غير العادية\n- تدقيق دوري للسجلات\n```\n\n### الكشف عن التهديدات / Threat Detection\n\n```\n- فحص دوري للثغرات الأمنية\n- اختبارات الاختراق المنتظمة\n- تحديثات أمنية فورية\n- فريق أمان مخصص\n```\n\n---\n\n## 9. استجابة الحوادث / Incident Response\n\n### خطة الاستجابة / Response Plan\n\n1. **الكشف**: مراقبة 24/7 للأنشطة المريبة\n2. **التصرف**: إجراء فوري لعزل المشكلة\n3. **التحقيق**: تحليل شامل للحادث\n4. **الإبلاغ**: إخطار المستخدمين المتأثرين\n5. **الإصلاح**: تصحيح الثغرة ومنع تكرارها\n\n---\n\n## 10. سياسة الخصوصية / Privacy Policy\n\n### ما الذي نجمعه / What We Collect\n\n```\n- بيانات الحساب (البريد الإلكتروني، الاسم)\n- بيانات المنتجات والطلبات (من التكاملات)\n- بيانات الحملات الإعلانية (من التكاملات)\n- سجلات الاستخدام والأنشطة\n```\n\n### كيف نستخدمها / How We Use Them\n\n```\n✅ لتحسين الخدمة\n✅ لتقديم التوصيات\n✅ لتحليل الأداء\n✅ لأغراض أمنية\n❌ لا نبيع البيانات\n❌ لا نشاركها مع أطراف ثالثة\n```\n\n### مشاركة البيانات / Data Sharing\n\n```\n- لا نشارك البيانات مع أطراف ثالثة\n- المشاركة فقط مع مزودي الخدمات الضروريين\n- اتفاقيات حماية بيانات صارمة\n```\n\n---\n\n## 11. الامتثال والتدقيق / Compliance & Auditing\n\n### التدقيق الدوري / Regular
